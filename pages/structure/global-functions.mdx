# GlobalFunction

This PHP script provides a class-based implementation for determining the correct spelling of "summarizer" and "summarize" based on the current locale of the WordPress site. It uses the `get_locale()` function and checks for US English or other English variants.

## Class: `AISummarizer_SpellingHelper`

The class `AISummarizer_SpellingHelper` contains two static methods to handle locale-based spelling variations. These methods ensure that the appropriate spelling is used based on the site's locale settings.

## Methods

### `AISummarizer_getSummarizerSpelling()`

**Description**:
Determines the correct spelling of the word "summarizer" based on the current locale

**Return Values**:

-   **`string`**:
    -   Returns **`'Summarizer'`** for US English locales (e.g., `en_US`).
    -   Returns **`'Summariser'`** for all other locales (e.g., UK English, Australian English).

**Usage**:

```php
use AISummarizer\GlobalFunction\AISummarizer_SpellingHelper;

$summarizer_spelling = AISummarizer_SpellingHelper::AISummarizer_getSummarizerSpelling();
echo $summarizer_spelling; // Outputs 'Summarizer' or 'Summariser' based on locale

```

### `AISummarizer_getSummarizeSpelling()`

**Description**:
Returns the correct spelling of the word "summarize" based on the current locale.

**Return Values**:

-   **`string`**:
    -   Returns **`'Summarize'`** for US English locales (e.g., `en_US`).
    -   Returns **`'Summarise'`** for all other locales (e.g., UK English, Australian English).

**Usage**:

```php
use AISummarizer\GlobalFunction\AISummarizer_SpellingHelper;

$summarize_spelling = AISummarizer_SpellingHelper::AISummarizer_getSummarizeSpelling();
echo $summarize_spelling; // Outputs 'Summarize' or 'Summarise' based on locale

```

## How It Works

-   **Namespace**:
    The class is part of the `AISummarizer\GlobalFunction` namespace, ensuring compatibility and avoiding conflicts in the global scope.

-   **Locale Detection**:
    Each function uses the `get_locale()` function provided by WordPress to determine the current site's locale. The locale is typically in the format of a language code followed by an optional country code (e.g., en_US, en_GB, en_AU).

-   **Conditional Logic**:
    Each method checks if the locale starts with en_US. If true, the US English spelling is returned. For all other locales, the British English spelling is used.

-   **Integration**: These methods are designed to be used throughout your plugin, ensuring consistent locale-based spelling for user-facing text.
